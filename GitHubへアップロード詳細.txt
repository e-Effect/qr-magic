================================================================================
  GitHub の「QRマジック」リポジトリに、PCのファイルを上げる（詳しい手順）
  ※ ブラウザだけでできる方法（Git コマンド不要）
================================================================================

【この手順の目的】
  デスクトップのフォルダ
    C:\Users\user\Desktop\qr-magic
  にある「最新のファイル」を、GitHub 上の QRマジック リポジトリに反映します。
  そうすると Render が次回ビルドでその内容を取り込みます。

【絶対にアップロードしないもの】
  ・node_modules フォルダ（中身ごと大きすぎるし不要）
  ・.env（秘密が入るので GitHub に上げない）

--------------------------------------------------------------------------------
  準備：エクスプローラーでファイルを用意する
--------------------------------------------------------------------------------

  1) キーボードの Windowsキー + E で「エクスプローラー」を開く

  2) アドレスバー（上のパスが書いてある所）をクリックして、次をコピーして貼り付け、Enter

        C:\Users\user\Desktop\qr-magic

  3) このフォルダの中に、次のファイルがあるか確認する

        server.js
        embedded-index.js   ← 名前を一字一句確認（ハイフンあり）
        Dockerfile

  4) もし embedded-index.js が無い場合
     ・そのフォルダで PowerShell を開き、次を実行（1行）

        npm.cmd run embed

     ・もう一度 embedded-index.js があるか確認する

--------------------------------------------------------------------------------
  方法A：ファイルを「1つずつ」GitHubのWebで上書きする（確実）
--------------------------------------------------------------------------------

  全体の流れ：
    GitHubで該当ファイルを開く → 鉛筆マークで編集 → 中身を全部置き換え → 保存

  --- server.js を上げる ---

  1) ブラウザで GitHub を開き、ログインする

  2) 画面上部の検索や一覧から、リポジトリ「QRマジック」（名前は近いもの）を開く

  3) ファイル一覧の中から「server.js」を探してクリック
     ・無い場合は「Add file」→「Upload files」で新規に置く方法は「方法B」を参照

  4) ファイル表示の右上あたりにある「鉛筆」アイコン（Edit this file／このファイルを編集）を押す

  5) PC側でメモ帳を開き、次のファイルを開く

        C:\Users\user\Desktop\qr-magic\server.js

  6) メモ帳で「すべて選択」（Ctrl+A）→「コピー」（Ctrl+C）

  7) ブラウザの GitHub の編集画面で、表示されている文字を「すべて選択」（Ctrl+A）→「貼り付け」（Ctrl+V）
     ※ 中身が PC の server.js と完全に同じになればよいです

  8) ページ下の「Commit changes」または「変更をコミット」を押す

     ・「Commit message」には例:  update server for Render
     ・「Commit directly to the main branch」など、そのまま main に入る方を選ぶ

  9) 少し待つと、一覧に戻る。server.js の「最終更新」が今の日時に近づいていれば成功しやすいです

  --- embedded-index.js を上げる ---

  10) 同じリポジトリのトップ（ファイル一覧）に戻る

  11) 一覧に「embedded-index.js」があるか見る

      【ある場合】
        ・そのファイルを開く → 鉛筆で編集 → PCの
            C:\Users\user\Desktop\qr-magic\embedded-index.js
          をメモ帳で開いて全部コピー → GitHub に貼り付け → Commit changes

      【無い場合】
        ・「Add file」→「Create new file」を選ぶ
        ・「Name your file…」と書かれた入力欄に、次のように打つ（フォルダ名は付けない）

            embedded-index.js

        ・その下の大きな入力欄に、PCの embedded-index.js の中身を全部コピーして貼り付ける
        ・Commit changes

  --- Dockerfile を上げる ---

  12) server.js と同じ要領で、Dockerfile を開いて編集し、PCの

        C:\Users\user\Desktop\qr-magic\Dockerfile

      の中身で全部置き換え → Commit changes

--------------------------------------------------------------------------------
  方法B：複数ファイルをまとめて「Upload」する（慣れている人向け）
--------------------------------------------------------------------------------

  1) リポジトリのトップで「Add file」→「Upload files」

  2) エクスプローラーから、次の3つをドラッグ＆ドロップする

        server.js
        embedded-index.js
        Dockerfile

     ※ node_modules はドロップしない

  3) ページ下で「Commit changes」

  注意:
    ・すでに同じ名前のファイルがあると「置き換え」になることが多いです
    ・もし「コンフリクト」などと出たら、方法Aの「1つずつ上書き」に切り替えてください

--------------------------------------------------------------------------------
  public フォルダはどうする？
--------------------------------------------------------------------------------

  ・embedded-index.js を上げれば、**public が無くてもトップは動く**ようにしてあります
  ・余裕があれば、PCの

        C:\Users\user\Desktop\qr-magic\public

    フォルダ内のファイル（index.html など）も GitHub に「public」という名前のフォルダを作って
    同じ構造で置いておくと、あとで管理しやすいです（必須ではありません）

--------------------------------------------------------------------------------
  上げ終わったあと（Render）
--------------------------------------------------------------------------------

  1) Render のダッシュボードで、このアプリのサービスを開く

  2) 「Manual Deploy」→「Clear build cache & deploy」を実行

  3) ログが「Live」になるまで待つ

  4) ブラウザで URL を開き直す

--------------------------------------------------------------------------------
  うまくいったか GitHub だけで確認する
--------------------------------------------------------------------------------

  1) リポジトリの「Code」タブのファイル一覧で、embedded-index.js をクリック

  2) 表示が「Raw」ボタン付きの長い1行や、先頭が

        /** Auto-generated from public/index.html

     のようなコメントから始まる巨大なファイルなら、中身は載っています

  3) server.js を開き、検索（Ctrl+F）で「embedded-index」と入力
     ・「require("./embedded-index.js")」のような文字があれば、新しい server.js が載っています

================================================================================
  ユーザー名が「user」ではない場合
================================================================================

  デスクトップの場所は例えば次のようになります（あなたのWindows名に置き換え）:

        C:\Users\（あなたの名前）\Desktop\qr-magic
